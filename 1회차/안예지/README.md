````
레인지 자료형도 스텝이 가능해서 신기했다. 활용해 본 건 처음이어서 기억에 남는 문제 풀이였다. 더 줄여볼 수 있을 것 같은데(불필요한 변수 할당 등) 값 오류라도 날까 봐 허겁지겁 다음 문제로 넘어갔다. 코드리뷰 때 많은 조언을 들었으면 좋겠다.
````

### 신용카드 만들기 -2

````
고백하자면, if문에 대한 이해가 아직 덜 됐음을 여실하게 보여준 문제였다. 주말동안 if문 개념 복습을 해야겠다. 만약 지금보다 더 잘 알았다면 코드도 직관보다는 논리에 가까웠을 것 같다는 아쉬움이 남는다.
````

### 암호문 - 1

````
머리가 나쁘면 몸이 고생한다.
딕셔너리로 풀고 싶어서 이틀을 내리 고민했다. 구글링을 하지 않은 이유는 바로 정답 코드가 나올 것 같아서였고, '딕셔너리 키-값 한 번에 한 칸씩 뒤로 옮기는 법' 등을 검색해도 결과는 나오지 않았다. 그렇게 옮겨봤자 새 값을 추가하는 데 명령어 목록에서 가져와야할 값을 접근하지 못해 이틀을 고민하게 만들었다.
질문을 위해 접근 방법을 정리하다가, 수 백 번의 '이렇게 하면 될 것 같은데?'의 굴레에 빠져 질문도 못했다.
딕셔너리로 풀겠다는 자료구조를 생각한 이유는 처음에 암호문이 새로 추가되면서, '갱신'된다고 생각했기 때문이었다. 단순히 '추가'만 생각했으면 인덱스는 고정이라 보다 쉽게 풀리는 문제였는데 말이다.
출력 부분에서도 골머리를 앓았다. 출력 예시처럼 어떻게 깔끔하게 표현할 수 있는지 한참 고민하다 결국 문자열 더하기를 쓰는 수밖에 없었다.
리스트로 풀어봐야겠다는 생각이 얼핏 스치고, 특정 인덱스에만 값을 추가하면 되는 것 아닌가, 란 생각이 번뜻 스쳤다. 덕분에 insert메소드를 활용해 답안을 제출했고 의외로 정답이었다. 수 백번을 쓰다 지우다 했던 논리가 맞긴 맞은 모양이었다. 솔직히 너무 시간 낭비했다. 노트 7여장을 이 한 문제로 채웠다. 이 시간에 내 수준에 맞는 기초 문제나 개념 복습이나 할 걸. 아집과 집착의 시간이었고, 뿌듯함보다는 왠지 허망함이 남는 문제였다. 그래도 계속 집착하게 만든 건 아마 문제가 재밌어서였던 것 같다. 재미는 있었다. 주말이 날아가서 문제지.😥
````
